For Loops

There are many programming situations where you may want to have a section of code execute several times. For instance, let’s say that we want to have a single, one measure rhythm play on track 1 for 8 measures. At first, we might take this approach to the problem:

from earsketch import *
init()
setTempo(120)

beat = "0-00-00-0+++0+0+"
drum = ELECTRO_DRUM_MAIN_BEAT_008

makeBeat(drum, 1, 1, beat)
makeBeat(drum, 1, 2, beat)
makeBeat(drum, 1, 3, beat)
makeBeat(drum, 1, 4, beat)
makeBeat(drum, 1, 5, beat)
makeBeat(drum, 1, 6, beat)
makeBeat(drum, 1, 7, beat)
makeBeat(drum, 1, 8, beat)

finish()

This will yield the desired results. However, there’s a sign that there is a better way of doing things here… the code is very repetitive and difficult to read. Because programmers are always looking for simpler and clearer ways of representing common situations, there are techniques to accomplish the same thing while typing less. In this case, it would be much simpler to use a for loop. The for loop structure makes the code appear much more readable and clearly indicates the logic. A for loop works by being given a certain range of values through which to operate. This range is described by the range() function.

The range() function most commonly takes in 2 parameters, a starting point and an ending point. The starting point is inclusive, meaning that the starting number is included within the range. For example, if the starting number is 1, 1 is included within the range that the function is describing. On the other hand, the ending point parameter is exclusive, meaning that the ending number is not included within the range being described by the function. To be familiar with this concept, let’s look at an example.

range(1,9)

The above call refers to the range of numbers: 1, 2, 3, 4, 5, 6, 7, 8. Notice that the starting point parameter, 1, is included within this list, whereas the ending point parameter, 9, is not. The starting point is inclusive and the ending point is exclusive.

range(3,7)

This call refers to this range of numbers: 3, 4, 5, 6. Again, 3 is inclusive and 7 is exclusive.

The range() function can also be implemented with only 1 parameter. If given a single parameter, range() will provide a range from 0 to that number. Remember, the ending point (the single parameter that was passed in) is still exclusive. Therefore, the following function call yields the range 0, 1, 2, 3, 4:

range(5)

Additionally, range() can be used with three parameters. In this case, the first two parameters are the starting and ending points and the third parameter is a step variable. The step variable tells the range on what interval you want it to work. You can use the third parameter to create ranges of numbers that increase by two or four or any other number. For example, the following function call yields the range 0, 3, 6, 9:

range(10,0,3)

The range() function is important within the use of loops because it provides a range of numbers for the loop to iterate through. Let’s look at how the range function can be used with loops by reconsidering the problem presented at the beginning of this section: place a rhythm on track 1 of an EarSketch project for 8 measures.

from earsketch import *

#setup
init()
setTempo(120)

#variables
drum = ELECTRO_DRUM_MAIN_BEAT_008
beat = "0-00-00-0+++0+0+"

# using iteration instead of repeating similar lines of code
for measure in range(1, 9):
    makeBeat(drum, 1, measure, beat)

#finish
finish()

The variable measure is the index variable for the loop. As the loop iterates, or executes the statements inside the loop, measure will be assigned numbers from 1 through 8, based on the range() function.

As measure increases, so does the location value within the makeBeat() function that places your rhythm onto the timeline in Reaper. This loop will effectively place one rhythm on the first 8 measures of track 1. Compare this code with the code at the beginning of this section — both accomplish the same task, but the for loop makes the code significantly shorter and easier to read.

Fills

Fills are components in music that can add some excitement to repetitive lines of music. This is a technique that is frequently implemented within drum tracks. For instance, there might be a repetitive drum beat like the one in our previous example. In order to make this repetitive loop more exciting, we might add a different beat on every fourth measure. This would create more interesting music where we have three measures of a consistent rhythm punctuated by some different rhythm.

In order to create a track with a consistent beat and a fill, we will use our knowledge of for loops and the range function. We will make use of the version of the range function that has three parameters (start, end, increment value) and a for loops to accomplish our task.

from earsketch import *

init()
setTempo(120)

beatPattern = '0-0-0-000+++0-0-'
fillPattern = '0++00++00+0+0000'

drum = ELECTRO_DRUM_MAIN_BEAT_001

for counter in range(1, 13, 4):
    # three measures of a standard pattern
    makeBeat(drum, 1, counter,  beatPattern)
    makeBeat(drum, 1, counter + 1, beatPattern)
    makeBeat(drum, 1, counter + 2, beatPattern)
    # one measure of a fill pattern
    makeBeat(drum, 1, counter + 3, fillPattern)

finish()

Copy and paste this code into a new EarSketch script, save it, run it, and listen to the results.

Exercises

1. Modify the example above to change the length of the drum pattern to 16 measures. Change the drum sound it uses and change the beat pattern strings it uses.

2. Change your drum track so that the fill now appears on every third measure.

 
 

Conditionals

Often, we will want the computer to be able to compare two values and perform an action based on the relationship between those two values. For example, if a measure number is divisible by 2 (i.e., if it’s an even measure), we might want to add percussion to the measure. Otherwise, we might want to add guitars to that measure. There are two steps in this process:

  • Check whether a variable has a particular relationship to a value or another variable, and
  • Perform an action (or a series of actions, or no action) based on the result of this check.

These two steps combine to form a conditional in computer programming.  This name comes from the idea that  the first step involves the computer checking whether or not a certain condition has been met. A conditional tells the computer to perform an action if the condition is met, and to perform a separate action if the condition is not met. (Keep in mind that either one of these actions can be to do nothing at all.)

We use conditionals in our everyday lives. Consider grocery shopping – how do you know whether or not you should purchase a carton of milk? Before you go shopping, you checkyour fridge to see if you already have milk. If you don’t, then you buy some at the store; if you do, then you don’t buy any. In this case, the condition you are checking is whether or not you have milk in your fridge, and the action you are performing is buying milk at the store.

The milk example illustrates an important point about conditionals – they can be written as “if-then” statements. For this example, the conditional would be phrased as: “If there is no milk in the fridge, then you should buy some at the store.” In programming, the “if” part of the conditional will typically involve a comparison. The “then” part is written below the “if” part and is always indented; if there are multiple actions to be performed within the “then” part, they should all be indented. Consider the following code:

checkValue = 1
if checkValue == 1:
    fitMedia(HIPHOP_SYNTH_LEAD_001, 1, 1, 3)

This illustrates the typical structure of a conditional.  Line 2 compares the value of checkValue to 1 to see if the two are equal. If they are, the “then” part of the conditional on line 3 is executed, and drums are added to track 1; if they aren’t, the “then” part of the conditional is not executed. If the value of checkValue were changed to 2, for example, nothing would happen – no audio clip would be added to the song.

Notice how the “if” statement in line 2 compares two values using the “==” sign (also called a “double equals” sign); this is different than the single “=” sign we use when assigning variables. When you see the “==” sign, it means the computer is performing a comparison. Other comparisons can be used in the “if” part of the conditional: greater than (>), less than (<), greater than or equal to (>=), less than or equal to (>=), and not equal to (!=). For example, if the second line in the previous example were changed to

checkValue = 1
if checkValue == 1:
    fitMedia(HIPHOP_SYNTH_LEAD_001, 1, 1, 3)
